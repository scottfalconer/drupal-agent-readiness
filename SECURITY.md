# Security

This repo contains public benchmark/evidence artifacts and a prototype Drupal
module. It is not production software.

Do not include private site data, secrets, credentials, access tokens, database
dumps, or customer transcripts in issues, pull requests, run artifacts, or
fixtures.

For a vulnerability in the prototype module or scripts, open a private report
through GitHub's vulnerability reporting flow if it is enabled for the
repository. If private reporting is not available, file a minimal public issue
without exploit details and title it "Security contact request"; a maintainer
will provide a private contact path. The current maintainer contact on GitHub is
`@scottfalconer`.

For ordinary evaluator bugs, documentation issues, or claim-boundary concerns,
use a normal GitHub issue.
